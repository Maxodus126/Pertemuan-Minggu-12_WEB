ℹ️ Cara Menggunakan

Klik salah satu kartu role di atas atau gunakan form login di sidebar. Setiap role memiliki tampilan data berbeda — admin melihat semua, mahasiswa hanya data sendiri, dosen lihat data mahasiswa dengan masking, pimpinan lihat ringkasan.

📖 Konsep Security & Governance:
Aplikasi data harus aman, terkontrol, dan dapat diaudit. Setiap role mendapat tampilan data berbeda. Data sensitif dimasking otomatis. Setiap aktivitas tercatat di audit_log dengan timestamp. Backup & DR plan memastikan keberlangsungan sistem.

📖 Konsep Security & Governance (Modul 12):
RBAC: Role-Based Access Control — setiap user memiliki hak akses sesuai rolnya.
Masking: Data sensitif (email, no_hp) disembunyikan untuk role tertentu.
Audit Log: Setiap aktivitas dicatat dengan user, role, aksi, dan timestamp.
Backup & DR: Rencana pemulihan data dengan RTO dan RPO yang terukur.
